# 太阳能供电同步闪烁路灯 - 规格书

> 版本：v2.0.0 | 创建：2025-03-09 | 更新：2026-03-24
> **变更说明**：MCU 从 STM32F103ZET6 移植到 STM32C011F6P6，IO 重映射，新增详细 IO 状态表

---

## 1. 项目概述

太阳能供电的同步闪烁路灯系统，白天由太阳能板为镍氢电池充电，夜间自动检测并开启同步闪烁照明。

---

## 2. MCU 选型

| 阶段 | 型号 | 封装 | 说明 |
|------|------|------|------|
| 开发期 | STM32F103ZET6 | LQFP144 | 现有开发板，便于快速开发 |
| 量产期 | **STM32C011F6P6** | **TSSOP-20** | 低成本，32KB Flash，支持 Stop/Sleep 低功耗，**已移植完成** |

**软件移植状态**：已完成从 F103 到 C011F6P6 的完整移植，所有功能已验证。

---

## 3. 无线同步

- **模块**：深圳市芯岭技术 XL2400T（2.4G）
- **方式**：对等同步（无主节点，各模块硬件一致）
  - 每周期约 50ms 前进入短 RX 窗口；收到 sync 则对齐定时器且不发送；未收到且本机到点则发送 sync 并闪灯
  - 自然涌现发起者，每周期仅 1 次 TX，其余只 RX
- **规模**：几百个、直线/堆叠、间距几米、单跳几十米 → 邻节点重叠，sync 逐跳传播，5～10 周期内整网收敛

---

## 4. 电源系统

| 项目 | 规格 |
|------|------|
| 电池 | 1.2V 镍氢电池，800mAh |
| 太阳能板 | 2.4V 太阳能电池板 |
| 供电链路 | 1.2V NiMH → DC-DC 升压至 3.3V (MCU) / LED 升压驱动 |
| 充电 | 太阳能板直充 + MOSFET 控制，充满后关断 |

**过放/过充与滞回：**

| 项目 | 阈值 | 动作 |
|------|------|------|
| 过放 | < 1.0V | **由升压芯片 ME2188A 的 0.9V 启动阈值 + 整机掉电实现过放保护，MCU 不做额外关机逻辑** |
| 过放恢复 | > 1.1V | 白天由太阳能板重新启动升压芯片，MCU 上电后恢复正常工作 |
| 过充 | ≥ 1.55V | 关断充电 MOSFET（参考镍氢规格书快充末端电压） |
| 充电滞回 | < 1.35V | 重新开启充电（避免边界抖动） |

## 5. 日/夜检测

| 项目 | 规格 |
|------|------|
| 检测对象 | 太阳能板输出电压 (PA0) |
| 开启 | < 0.29V → 开启闪灯 |
| 关闭 | > 0.4V → 关闭闪灯 |
| 说明 | 滞回 0.11V，避免临界抖动 |

**采样方式**：太阳能板电压 0.29V～0.4V 直接接入 ADC，不分压、不放大（低成本）。

## 7. LED 驱动规格（v1.2.0 更新）

### 7.1 LED 配置
- **数量**：6 颗白色 LED 并联
- **型号**：TJ-S3014UG

### 7.2 驱动方式：电感升压驱动

| 参数 | 值 |
|------|-----|
| 升压电感 | L1 = 33μH |
| 开关管 | Q3 = CJ2302 (N-MOS) |
| PWM 频率 | **124kHz** |
| PWM 占空比 | **60%** |

### 7.3 控制引脚

| 引脚 | 名称 | 功能 | 控制逻辑 |
|------|------|------|----------|
| PA2 | LED_DRV | PWM 输出驱动升压 | 输出 PWM → LED 亮；LOW → LED 灭 |
| PA3 | BOOST_EN | 升压电路供电使能 | MCU 启动后 100ms 置 HIGH，始终保持 |

### 7.4 LED 控制时序

```
MCU 启动：
  初始化 → 时钟稳定 → 延时 100ms → PA3 (BOOST_EN) = HIGH

闪烁周期 (900ms)：
  [0-100ms]   PA2 输出 124kHz 60% PWM → LED 亮
  [100-900ms] PA2 = LOW (停止 PWM)    → LED 灭
  
PA3 始终保持 HIGH（不随 LED 亮灭变化）
```

## 8. 低功耗策略

| 项目 | 措施 |
|------|------|
| 电池电压检测 | PA1 (ADC) 检测，预留功能 |
| 充电 | MOSFET 控制，充满关闭 |
| MCU | 闪灯间隙 Stop/Sleep |
| XL2400T | 非收发时休眠（约 1.53μA） |
| LED 升压 | 灭灯时停止 PWM，升压电路不工作 |
| 目标 | 压低整机平均电流，应对连续多日阴天 |

### 8.1 欠压深度低功耗（Standby + RTC 周期唤醒）经验

在“电池欠压/或人为将 PA1 拉低”的极端工况下，为了避免继续放电并压低静态电流，MCU 可进入 **Standby**，并通过 **RTC Alarm A** 周期唤醒（Standby exit 会导致复位启动）来周期性重测电池电压。

**重要结论（实测）**：
- XL2400T 即使已经通过 `RF_Link_Sleep()` 进入低功耗，**进入 Standby 后如果 RF 三线（CSN/SCK/DATA）在 Standby 期间变为高阻/漂移，RF 仍可能被异常边沿唤醒**，从而出现“Standby 反而 600µA”的假象。
- 解决关键是 Standby 期间的 IO 保持：使用 PWR 的 Pull-up/Pull-down 并启用 APC，使 Standby 内依然维持：  
  - `RF_CSN` 上拉为高  
  - `RF_SCK`、`RF_DATA` 下拉为低  

详细的踩坑清单与验证方法见：`docs/XL2400T_STANDBY_TRAPS_AND_TIPS.md`。

### 11.1 PA1(ADC1_IN1) 过充/过放联合保护评估结论

- **过充保护（可行）**：PA1 直采电池电压已具备可行性，现有策略为 `>=1.55V` 关断充电、`<1.35V` 重开，建议继续沿用。
- **过放保护设为 0.9V（不建议由 MCU ADC 直接执行）**：
  - 当电池逼近 0.9V 时，ME2188A 升压链路已接近失效，MCU 可能先掉电，无法稳定完成 ADC 采样与控制动作。
  - 因此 `0.9V` 更适合作为**硬件兜底阈值**（由升压芯片欠压特性实现），不应作为 MCU 软件“主动关机阈值”。
- **建议方案**：采用“软件预保护 + 硬件兜底”双层策略。

### 11.2 PA1 保护策略待办（建议写入开发任务）

1. 保持硬件兜底：继续以 ME2188A 欠压特性作为最终过放保护（约 0.9V 附近掉电）。
2. 新增软件预过放阈值：在 PA1 增加 `1.05V~1.10V` 预关断阈值（建议初值 1.08V），并加入 50~100mV 滞回。
3. 预过放动作：触发后关闭高耗电负载（LED/RF），仅保留最低功耗与周期唤醒检测（或直接进入更深低功耗）。
4. 充电恢复判据：电压回升到恢复阈值（如 1.15V 以上）再退出预过放状态。
5. 标定与容差：加入 ADC 校准、样本平均与温漂/批次容差统计，避免阈值误判。
6. 台架验证：覆盖常温/低温、弱光慢充、连续阴天场景，确认不会在阈值附近振荡。

## 12. 生产测试模式（规划，最后实现）

> 目的：生产时快速验证 LED/无线/ADC/IO，不受 3s/5s 采样确认等“慢逻辑”影响，提高产线效率。

### 12.1 进入方式（建议）

- 上电后前几秒检测“夹具专用条件”进入测试模式，避免现场误触发，例如：
  - 预留测试脚/某个 IO 被夹具强制拉低（或短接到 GND）
  - 或串口收到特定命令（仅产线使用）

### 12.2 测试内容（建议）

- 打印固件版本、UID、关键 ADC 原始值（Solar PA0 / Batt PA1）
- 快速点亮/熄灭 LED（验证 LED_DRV/升压）
- 快速发送/接收若干帧 2.4G（验证 XL2400T）
- 不执行正常模式下的长时间确认（如日夜 3s、过充二次确认等），仅用于“快速判定/打印结果”

### 12.3 退出方式（建议）

- 测试结束自动复位或提示断电重启，重启即回到正常模式
