# XL2400T 模式切换修正说明

> v1.3.1 | 2025-03-12

---

## 问题

RX 板的 DATA 引脚持续输出波形，说明未正确进入接收模式。

---

## 原因

原驱动从 py32f0-template 移植，`SetRxMode` 写的是：

```c
cbuf[0] = 0xee;
cbuf[1] = 0xc0;
```

这对 XL2400P 可能有效，但 XL2400T 的 `CFG_TOP` 寄存器布局略有不同。

---

## 修正

按 XL2400T 手册，`CFG_TOP` 寄存器关键位：

- bit 1 `PWR_ON`：1=上电工作，0=掉电
- bit 0 `RX_ON`：1=接收模式，0=发送模式

修正后：

- **TX 模式**：`0x0E 0x00`（PWR_ON=1, RX_ON=0, 其他位保持）
- **RX 模式**：`0x0F 0x00`（PWR_ON=1, RX_ON=1, 其他位保持）

---

## 测试

重新编译、烧录 TX/RX 板后，RX 板 DATA 应在空闲时无波形（高阻），收到包时才有短暂输出。
