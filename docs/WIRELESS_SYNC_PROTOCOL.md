# 无线同步协议 - 详细设计

> 版本：v1.0 | 适用于 XL2400T 对等同步

---

## 1. 概述

- **目标**：多盏路灯在 900ms 周期内实现同步闪烁（100ms 亮 / 800ms 灭）
- **方式**：对等同步，无主从，各模块硬件一致
- **芯片**：XL2400T，2.4G，250Kbps

---

## 2. 包格式

| 字段 | 长度 | 说明 |
|------|------|------|
| 地址 | 5 字节 | 全网统一 |
| payload | 8~16 字节 | sync 包内容 |

** sync payload 格式：**

| 偏移 | 长度 | 内容 | 说明 |
|------|------|------|------|
| 0 | 2 | 0xAA 0x55 | 帧头 |
| 2 | 2 | cycle_count | 周期计数（大端） |
| 4 | 1 | crc8 | 可选，简化时可省 |

**地址**：`{0x58, 0x4C, 0x32, 0x34, 0x30}`（ASCII "XL240"）

---

## 3. RF 参数

| 项目 | 值 |
|------|-----|
| 速率 | 250Kbps |
| channel | 0（可配置 0~80） |
| 功率 | 0dBm 或 -12dBm（低功耗） |
| ACK | 关闭（W_TX_PAYLOAD_NOACK），减少功耗 |

---

## 4. 对等同步流程

```
周期 T=0: 闪灯开始
    ├── 850ms: 唤醒 MCU，XL2400T 进入 RX
    ├── 850~900ms: RX 窗口（约 50ms）
    │   ├── 若收到 sync → 对齐本地定时器，本周期不发送
    │   └── 若未收到且本机到点 → 发送 sync 并闪灯（本周期发起者）
    ├── 900ms: 闪灯（100ms）
    ├── 1000ms: 灭灯
    └── 1000~1900ms: Sleep（MCU + XL2400T 休眠）
```

**规则：**

1. 收到 sync → 立即对齐 900ms 定时器为 0，不发送
2. 未收到 sync 且本机 900ms 定时器到点 → 发送 sync，然后闪灯
3. 每次 payload 必须包含变化字节（cycle_count），避免相同包被芯片丢弃

---

## 5. 时序示意

```
Node A:  [======Sleep======][RX 50ms][TX sync][Flash 100ms][======Sleep======]
Node B:  [======Sleep======][RX 50ms]         [Flash 100ms][======Sleep======]
                              ↑ 收到 A 的 sync，对齐
```

---

## 6. 实现要点

- **RX 窗口**：约 50ms，足够覆盖传播延迟和时钟偏差
- **收敛**：邻节点重叠覆盖，sync 逐跳传播，约 5~10 周期内整网同步
- **功耗**：非 RX 时段 XL2400T 休眠（约 1.53μA）
