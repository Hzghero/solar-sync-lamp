# 初步通讯验证清单

## 前置条件

- [ ] STM32F103ZET6 开发板
- [ ] XL2400T 模块，已按接线表连接
- [ ] STM32CubeMX 工程已生成，SPI1 + USART1 已配置

## 步骤 1：SPI 自检

1. 将驱动与 port 文件加入工程
2. 在 `main()` 中调用 `XL2400_SPI_Test()`
3. 若返回 `XL2400_OK`，说明 SPI 通讯正常

## 步骤 2：收发环回

1. 准备两块板（或两块开发板 + 两个 XL2400T 模块）
2. 板 A：`XL2400_APP_MODE = 1`（TX），编译烧录
3. 板 B：`XL2400_APP_MODE = 0`（RX），编译烧录，串口打印接收
4. 上电后，板 B 串口应能周期性收到板 A 发送的数据

## 通过标准

- SPI 自检通过
- 双板环回时，RX 端能稳定收到 TX 端数据
